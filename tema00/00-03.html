Instalación de WSL

Windows Subsystem for Linux - Un entorno Linux completo en Windows

¿Qué es WSL?

Windows Subsystem for Linux (WSL) es una característica de Windows 10 y Windows 11 que permite ejecutar un entorno Linux directamente en Windows, sin necesidad de una máquina virtual tradicional o configuración de arranque dual.

WSL te proporciona:

  • Un kernel de Linux completo ejecutándose en Windows
  • Acceso a herramientas y aplicaciones de línea de comandos de Linux
  • Integración perfecta con el sistema de archivos de Windows
  • Capacidad de ejecutar aplicaciones Linux nativas
  • Mejor rendimiento que las máquinas virtuales tradicionales
¿Por qué usar WSL para Python?

Muchas herramientas de desarrollo y librerías de Python están diseñadas para entornos Unix/Linux. WSL te permite usar estas herramientas en Windows con la máxima compatibilidad.

WSL 1 vs WSL 2

  • WSL 1: Capa de compatibilidad que traduce llamadas del sistema Linux a Windows
  • WSL 2: Usa un kernel de Linux real en una máquina virtual ligera optimizada

Para desarrollo con Python, recomendamos WSL 2 por su mejor rendimiento y compatibilidad completa.

Requisitos del Sistema

Para instalar WSL 2, tu sistema debe cumplir con los siguientes requisitos:

Para Windows 10

  • Versión: Windows 10 versión 1903 o superior, con Build 18362 o superior
  • Para sistemas x64: Versión 1903 o superior, con Build 18362 o superior
  • Para sistemas ARM64: Versión 2004 o superior, con Build 19041 o superior
  • Virtualización: Debe estar habilitada en la BIOS

Para Windows 11

  • WSL 2 está totalmente soportado en todas las versiones de Windows 11
  • Virtualización: Debe estar habilitada en la BIOS
Verificar la versión de Windows

Puedes verificar tu versión de Windows ejecutando winver en el menú Inicio o usando Configuración > Sistema > Acerca de.

Verificar si la virtualización está habilitada

Para comprobar si la virtualización está habilitada:

  1. Abre el Administrador de tareas (Ctrl + Shift + Esc)
  2. Ve a la pestaña "Rendimiento"
  3. Selecciona "CPU"
  4. Busca "Virtualización: Habilitada" en la parte inferior derecha
Si la virtualización está deshabilitada

Si la virtualización está deshabilitada, necesitarás habilitarla en la BIOS. El proceso varía según el fabricante de tu motherboard. Busca opciones como "Intel VT-x", "AMD-V", o "SVM" en la configuración de la BIOS.

Método 1: Instalación Simple (Windows 10 versión 2004+)

Si tienes Windows 10 versión 2004 o superior, o Windows 11, puedes instalar WSL con un solo comando:

Instalación con un comando PowerShell (Admin) 
# Abre PowerShell como Administrador y ejecuta:
wsl --install

Este comando:

  • Habilita las características necesarias de Windows
  • Descarga e instala el kernel de Linux más reciente
  • Establece WSL 2 como versión predeterminada
  • Instala Ubuntu como distribución predeterminada

Instalar una distribución específica

Si prefieres una distribución diferente a Ubuntu:

Ver distribuciones disponibles PowerShell 
# Ver distribuciones disponibles:
wsl --list --online

# Instalar una distribución específica:
wsl --install -d Ubuntu-22.04
wsl --install -d Debian
wsl --install -d openSUSE-Leap-15.4
Reinicio necesario

Después de ejecutar wsl --install, necesitarás reiniciar tu computadora para completar la instalación.

Método 2: Instalación Manual (Windows 10 versiones anteriores)

Si tienes una versión anterior de Windows 10, necesitarás instalar WSL manualmente siguiendo estos pasos:

Paso 1: Habilitar WSL

Abre PowerShell como Administrador y ejecuta:

Habilitar WSL PowerShell (Admin) 
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart

Paso 2: Habilitar la característica de Máquina Virtual

Para WSL 2, también necesitas habilitar la plataforma de máquina virtual:

Habilitar Plataforma de Máquina Virtual PowerShell (Admin) 
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart

Paso 3: Reiniciar el sistema

Reinicia tu computadora para aplicar los cambios.

Paso 4: Descargar el paquete de actualización del kernel de Linux

Descarga e instala el paquete de actualización del kernel de WSL 2:

  • Ve a: https://aka.ms/wsl2kernel
  • Descarga el archivo "WSL2 Linux kernel update package for x64 machines"
  • Ejecuta el archivo descargado e instálalo

Paso 5: Establecer WSL 2 como versión predeterminada

Configurar WSL 2 por defecto PowerShell 
wsl --set-default-version 2

Paso 6: Instalar una distribución de Linux

Puedes instalar una distribución desde Microsoft Store o usando PowerShell:

Instalar Ubuntu desde PowerShell PowerShell 
# Ver distribuciones disponibles:
wsl --list --online

# Instalar Ubuntu (recomendado para principiantes):
wsl --install -d Ubuntu

Configuración Inicial de la Distribución

Después de instalar tu distribución de Linux, necesitarás configurarla:

  1. Abrir la distribución: Busca el nombre de tu distribución en el menú Inicio (ej: "Ubuntu") y ábrela
  2. Esperar la instalación: La primera vez tardará unos minutos en completar la instalación
  3. Crear usuario: Se te pedirá crear un nombre de usuario y contraseña
  4. Actualizar el sistema: Es recomendable actualizar los paquetes del sistema
Configuración inicial de Ubuntu Linux Terminal 
# Actualizar la lista de paquetes:
sudo apt update

# Actualizar todos los paquetes instalados:
sudo apt upgrade -y

# Instalar herramientas esenciales:
sudo apt install -y curl wget git build-essential
Consejos para el usuario y contraseña

El nombre de usuario no tiene que coincidir con tu usuario de Windows. La contraseña se usará para comandos sudo, así que asegúrate de recordarla.

Verificar la Instalación de WSL

Para verificar que WSL se instaló correctamente, puedes usar estos comandos:

Verificar WSL desde PowerShell PowerShell 
# Ver todas las distribuciones instaladas:
wsl --list --verbose

# Ver la versión de WSL:
wsl --version

# Acceder a la distribución predeterminada:
wsl

La salida de wsl --list --verbose debería mostrar algo similar a:

Salida esperada Output 
  NAME      STATE           VERSION
* Ubuntu    Running         2
Verificar la versión

Asegúrate de que la columna VERSION muestre "2". Si muestra "1", puedes convertir la distribución a WSL 2 con: wsl --set-version Ubuntu 2

Comandos Útiles de WSL

Aquí tienes algunos comandos útiles para administrar WSL:

Comandos básicos de administración PowerShell 
# Listar distribuciones instaladas:
wsl --list

# Iniciar una distribución específica:
wsl -d Ubuntu

# Apagar todas las distribuciones:
wsl --shutdown

# Apagar una distribución específica:
wsl --terminate Ubuntu

# Establecer distribución predeterminada:
wsl --set-default Ubuntu

# Desinstalar una distribución:
wsl --unregister Ubuntu

Acceso a archivos entre Windows y WSL

Una de las ventajas de WSL es la integración con el sistema de archivos de Windows:

  • Desde WSL acceder a Windows: Los discos de Windows están montados en /mnt/ (ej: /mnt/c/ para el disco C:)
  • Desde Windows acceder a WSL: Usa la ruta \\wsl$\NombreDistribucion\ en el Explorador de archivos
Ejemplos de navegación entre sistemas Linux Terminal 
# Desde WSL, acceder al escritorio de Windows:
cd /mnt/c/Users/TuUsuario/Desktop

# Ver el contenido del disco C: desde WSL:
ls /mnt/c/

# Crear un archivo en WSL que sea visible desde Windows:
echo "Hola desde WSL" > ~/archivo_wsl.txt
Importante sobre los archivos

Para mejor rendimiento, mantén los archivos del proyecto en el sistema de archivos de Linux (dentro de WSL) cuando trabajes principalmente desde Linux, y en Windows cuando trabajes desde Windows.
