Python en WSL

Instalación y configuración de Python en el entorno Linux de WSL

Verificar Python Preinstalado

La mayoría de distribuciones de Linux vienen con Python preinstalado. Primero verificaremos qué versiones están disponibles en tu sistema WSL:

Verificar versiones de Python Linux Terminal 
# Verificar si Python 3 está instalado:
python3 --version

# Verificar si Python 2 está instalado (no recomendado):
python --version

# Ver la ubicación de Python 3:
which python3

# Ver todas las versiones disponibles:
ls /usr/bin/python*

Si Python 3 está instalado, deberías ver una salida similar a:

Salida esperada Output 
$ python3 --version
Python 3.10.12

$ which python3
/usr/bin/python3
Diferencia entre python y python3

En muchas distribuciones Linux, python puede apuntar a Python 2.x y python3 a Python 3.x. Siempre usa python3 para asegurarte de usar Python 3.

Instalar Python 3 (si no está disponible)

Si Python 3 no está instalado o quieres instalar una versión más reciente, puedes hacerlo usando el gestor de paquetes de tu distribución:

En Ubuntu/Debian

Instalar Python en Ubuntu/Debian Linux Terminal 
# Actualizar la lista de paquetes:
sudo apt update

# Instalar Python 3 y herramientas esenciales:
sudo apt install -y python3 python3-pip python3-venv python3-dev

# Instalar una versión específica (ejemplo Python 3.11):
sudo apt install -y python3.11 python3.11-venv python3.11-dev

# Verificar la instalación:
python3 --version
pip3 --version

En otras distribuciones

Instalar Python en otras distribuciones Linux Terminal 
# En CentOS/RHEL/Fedora:
sudo dnf install python3 python3-pip python3-venv

# En openSUSE:
sudo zypper install python3 python3-pip python3-venv

# En Arch Linux:
sudo pacman -S python python-pip
Paquetes recomendados

Siempre instala junto con Python: python3-pip (gestor de paquetes), python3-venv (entornos virtuales) y python3-dev (headers para compilar extensiones).

Configurar alias para Python

Para facilitar el uso de Python, es recomendable configurar alias que te permitan usar python en lugar de python3:

Crear alias temporales Linux Terminal 
# Crear alias temporal para la sesión actual:
alias python=python3
alias pip=pip3

# Verificar que funciona:
python --version
pip --version

Para hacer estos alias permanentes, añádelos a tu archivo de configuración del shell:

Crear alias permanentes Linux Terminal 
# Editar el archivo .bashrc (para bash):
nano ~/.bashrc

# O editar .zshrc (si usas zsh):
nano ~/.zshrc

# Añadir al final del archivo:
alias python=python3
alias pip=pip3

# Aplicar los cambios:
source ~/.bashrc
# o
source ~/.zshrc
Cuidado con los alias

Algunos scripts del sistema pueden depender del comando python apuntando a Python 2. Si experimentas problemas, puedes remover los alias temporalmente.

Instalar versiones específicas de Python

Para desarrollo profesional, es común necesitar versiones específicas de Python. Te mostraremos cómo instalar múltiples versiones:

Método 1: Usando deadsnakes PPA (Ubuntu)

El repositorio deadsnakes contiene múltiples versiones de Python para Ubuntu:

Instalar desde deadsnakes PPA Ubuntu Terminal 
# Añadir el repositorio deadsnakes:
sudo add-apt-repository ppa:deadsnakes/ppa

# Actualizar la lista de paquetes:
sudo apt update

# Instalar versiones específicas:
sudo apt install python3.9 python3.9-venv python3.9-dev
sudo apt install python3.10 python3.10-venv python3.10-dev
sudo apt install python3.11 python3.11-venv python3.11-dev
sudo apt install python3.12 python3.12-venv python3.12-dev

# Verificar las instalaciones:
python3.9 --version
python3.10 --version
python3.11 --version
python3.12 --version

Método 2: Compilar desde el código fuente

Para máximo control, puedes compilar Python desde el código fuente:

Compilar Python desde fuente Linux Terminal 
# Instalar dependencias de compilación:
sudo apt update
sudo apt install -y build-essential zlib1g-dev libncurses5-dev libgdbm-dev \
    libnss3-dev libssl-dev libreadline-dev libffi-dev libsqlite3-dev \
    wget libbz2-dev

# Descargar Python (ejemplo: versión 3.11.5):
cd /tmp
wget https://www.python.org/ftp/python/3.11.5/Python-3.11.5.tgz
tar -xf Python-3.11.5.tgz

# Compilar e instalar:
cd Python-3.11.5
./configure --enable-optimizations --prefix=/usr/local
make -j $(nproc)
sudo make altinstall

# Verificar la instalación:
python3.11 --version
make altinstall vs make install

Usa make altinstall en lugar de make install para evitar sobrescribir el Python del sistema. Esto instala Python como python3.11 en lugar de python3.

Gestión de versiones con pyenv

pyenv es una herramienta que facilita la instalación y cambio entre múltiples versiones de Python:

Instalar pyenv Linux Terminal 
# Instalar dependencias:
sudo apt update
sudo apt install -y make build-essential libssl-dev zlib1g-dev \
    libbz2-dev libreadline-dev libsqlite3-dev wget curl llvm \
    libncursesw5-dev xz-utils tk-dev libxml2-dev libxmlsec1-dev \
    libffi-dev liblzma-dev

# Descargar e instalar pyenv:
curl https://pyenv.run | bash

# Añadir pyenv al PATH (añadir al ~/.bashrc):
echo 'export PYENV_ROOT="$HOME/.pyenv"' >> ~/.bashrc
echo 'command -v pyenv >/dev/null || export PATH="$PYENV_ROOT/bin:$PATH"' >> ~/.bashrc
echo 'eval "$(pyenv init -)"' >> ~/.bashrc

# Recargar el shell:
exec $SHELL

Usar pyenv para instalar y gestionar versiones de Python:

Usar pyenv Linux Terminal 
# Ver versiones disponibles:
pyenv install --list

# Instalar versiones específicas:
pyenv install 3.9.18
pyenv install 3.10.13
pyenv install 3.11.5
pyenv install 3.12.0

# Ver versiones instaladas:
pyenv versions

# Establecer versión global:
pyenv global 3.11.5

# Establecer versión local para un proyecto:
mkdir mi_proyecto
cd mi_proyecto
pyenv local 3.10.13

# Verificar versión activa:
python --version
Ventajas de pyenv

pyenv permite cambiar fácilmente entre versiones de Python por proyecto, instalar versiones no disponibles en los repositorios oficiales, y mantener un entorno limpio sin interferir con el Python del sistema.

Configurar pip y herramientas de desarrollo

Una vez que tienes Python instalado, es importante configurar pip y instalar herramientas esenciales de desarrollo:

Actualizar pip y instalar herramientas Linux Terminal 
# Actualizar pip a la última versión:
python3 -m pip install --upgrade pip

# Instalar herramientas esenciales:
pip3 install --upgrade setuptools wheel

# Instalar herramientas de desarrollo recomendadas:
pip3 install virtualenv virtualenvwrapper ipython jupyter

# Verificar las instalaciones:
pip3 --version
virtualenv --version
ipython --version

Configurar virtualenvwrapper (opcional)

virtualenvwrapper facilita la gestión de entornos virtuales:

Configurar virtualenvwrapper Linux Terminal 
# Añadir al ~/.bashrc:
echo 'export WORKON_HOME=$HOME/.virtualenvs' >> ~/.bashrc
echo 'export VIRTUALENVWRAPPER_PYTHON=/usr/bin/python3' >> ~/.bashrc
echo 'source /usr/local/bin/virtualenvwrapper.sh' >> ~/.bashrc

# Recargar el shell:
source ~/.bashrc

# Crear un entorno virtual de prueba:
mkvirtualenv test_env

# Activar entorno:
workon test_env

# Desactivar entorno:
deactivate

# Listar entornos:
lsvirtualenv

Verificar la instalación completa

Para asegurarte de que todo está configurado correctamente, ejecuta estas verificaciones:

Script de verificación Linux Terminal 
# Crear un script de verificación:
cat << 'EOF' > verificar_python.py
#!/usr/bin/env python3

import sys
import platform
import subprocess

print("=== Verificación de Python en WSL ===")
print(f"Versión de Python: {sys.version}")
print(f"Plataforma: {platform.platform()}")
print(f"Arquitectura: {platform.architecture()}")
print(f"Ejecutable de Python: {sys.executable}")

# Verificar pip
try:
    import pip
    print(f"Versión de pip: {pip.__version__}")
except ImportError:
    print("pip no está disponible")

# Verificar módulos importantes
modules = ['venv', 'json', 'urllib', 'sqlite3', 'ssl']
for module in modules:
    try:
        __import__(module)
        print(f"✓ Módulo {module} disponible")
    except ImportError:
        print(f"✗ Módulo {module} NO disponible")

print("\n=== Verificación completada ===")
EOF

# Ejecutar el script:
python3 verificar_python.py

También puedes verificar que puedes crear y ejecutar un programa Python simple:

Programa de prueba Python 
# Crear un programa de prueba:
cat << 'EOF' > hola_wsl.py
#!/usr/bin/env python3

def main():
    print("¡Hola desde Python en WSL!")
    print("Python está funcionando correctamente.")
    
    # Prueba de importaciones básicas
    import os
    import datetime
    
    print(f"Sistema operativo: {os.name}")
    print(f"Fecha y hora: {datetime.datetime.now()}")
    
    # Prueba de operaciones básicas
    numeros = [1, 2, 3, 4, 5]
    suma = sum(numeros)
    print(f"Suma de {numeros} = {suma}")

if __name__ == "__main__":
    main()
EOF

# Ejecutar el programa:
python3 hola_wsl.py
Salida esperada

Si todo está configurado correctamente, deberías ver información sobre tu sistema Python y confirmaciones de que los módulos esenciales están disponibles.

Integración con editores de código

Para desarrollo productivo, es importante que tu editor de código pueda acceder al Python de WSL:

Visual Studio Code con WSL

VS Code tiene excelente integración con WSL:

  1. Instala la extensión "Remote - WSL" en VS Code
  2. Abre WSL y navega a tu directorio de proyecto
  3. Ejecuta code . para abrir VS Code conectado a WSL
  4. Instala la extensión de Python dentro del entorno WSL
Abrir VS Code desde WSL Linux Terminal 
# Crear un directorio de proyecto:
mkdir ~/mi_proyecto_python
cd ~/mi_proyecto_python

# Crear un archivo Python:
echo 'print("Hola desde WSL con VS Code!")' > main.py

# Abrir VS Code conectado a WSL:
code .
Configurar intérprete de Python en VS Code

En VS Code conectado a WSL, presiona Ctrl+Shift+P, busca "Python: Select Interpreter" y selecciona la versión de Python que quieres usar para tu proyecto.
