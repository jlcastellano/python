Entornos Virtuales en Python

Gestión y configuración de entornos aislados para proyectos Python

¿Qué son los Entornos Virtuales?

Un entorno virtual en Python es un directorio que contiene una instalación independiente de Python, junto con sus propios paquetes y dependencias. Esto permite tener proyectos completamente aislados unos de otros.

¿Por qué usar entornos virtuales?

  • Aislamiento de dependencias: Cada proyecto tiene sus propias versiones de librerías
  • Evitar conflictos: Diferentes proyectos pueden usar diferentes versiones de la misma librería
  • Reproducibilidad: Facilita recrear el mismo entorno en diferentes máquinas
  • Limpieza: No contaminas el Python del sistema con paquetes de proyectos específicos
  • Experimentación segura: Puedes probar nuevas librerías sin riesgo
Analogía útil

Piensa en los entornos virtuales como "cajas separadas" para cada proyecto. Cada caja contiene solo las herramientas específicas que necesita ese proyecto, sin interferir con otras cajas.

Problema sin entornos virtuales

Imagina que tienes dos proyectos:

  • Proyecto A: Necesita Django 3.2
  • Proyecto B: Necesita Django 4.2

Sin entornos virtuales, solo podrías tener una versión de Django instalada globalmente, causando problemas de compatibilidad.

Herramientas para Entornos Virtuales

Python ofrece varias herramientas para crear y gestionar entornos virtuales:

1. venv (Recomendado - incluido en Python 3.3+)

  • Viene integrado con Python
  • Simple y ligero
  • Recomendado para la mayoría de casos

2. virtualenv

  • Versión más antigua y con más características
  • Compatible con Python 2 y 3
  • Más opciones de configuración

3. conda

  • Gestor de paquetes y entornos
  • Especialmente popular en ciencia de datos
  • Puede gestionar dependencias no-Python

4. pipenv

  • Combina pip y virtualenv
  • Gestión automática de dependencias
  • Archivo Pipfile para especificar dependencias
¿Cuál elegir?

Para este curso y para empezar, recomendamos usar venv porque viene incluido con Python y es suficiente para la mayoría de proyectos. Una vez que te sientas cómodo, puedes explorar otras opciones.

Usando venv - Método Básico

venv es la herramienta estándar para crear entornos virtuales en Python 3.3+. Veamos cómo usarla:

En Windows

Crear y usar entorno virtual en Windows CMD/PowerShell 
# Crear un entorno virtual:
python -m venv mi_proyecto_env

# Activar el entorno virtual:
mi_proyecto_env\Scripts\activate

# Tu prompt cambiará a algo como:
(mi_proyecto_env) C:\ruta\del\proyecto>

# Verificar que estás en el entorno virtual:
where python
pip list

# Desactivar el entorno virtual:
deactivate

En Linux/WSL

Crear y usar entorno virtual en Linux/WSL Linux Terminal 
# Crear un entorno virtual:
python3 -m venv mi_proyecto_env

# Activar el entorno virtual:
source mi_proyecto_env/bin/activate

# Tu prompt cambiará a algo como:
(mi_proyecto_env) usuario@hostname:~/proyecto$

# Verificar que estás en el entorno virtual:
which python
pip list

# Desactivar el entorno virtual:
deactivate
Importante sobre la activación

Nota la diferencia en la activación: Windows usa Scripts\activate, mientras que Linux/WSL usa bin/activate. El comando de desactivación es el mismo en ambos sistemas.

Flujo de Trabajo Completo con venv

Veamos un flujo de trabajo completo para crear un proyecto Python con entorno virtual:

Flujo completo de proyecto Terminal 
# 1. Crear directorio del proyecto:
mkdir mi_nuevo_proyecto
cd mi_nuevo_proyecto

# 2. Crear entorno virtual:
python -m venv venv

# 3. Activar entorno virtual:
# En Windows:
venv\Scripts\activate
# En Linux/WSL:
source venv/bin/activate

# 4. Verificar que estamos en el entorno virtual:
python --version
pip --version

# 5. Actualizar pip (recomendado):
pip install --upgrade pip

# 6. Instalar paquetes necesarios:
pip install requests numpy pandas

# 7. Crear archivo de requerimientos:
pip freeze > requirements.txt

# 8. Ver contenido del archivo:
cat requirements.txt

El archivo requirements.txt contendrá algo similar a:

Ejemplo de requirements.txt Text 
certifi==2023.7.22
charset-normalizer==3.3.0
idna==3.4
numpy==1.25.2
pandas==2.1.1
python-dateutil==2.8.2
pytz==2023.3
requests==2.31.0
six==1.16.0
tzdata==2023.3
urllib3==2.0.5
Buenas prácticas

Siempre crea el archivo requirements.txt para documentar las dependencias de tu proyecto. Esto facilita que otros desarrolladores (o tú mismo en el futuro) puedan recrear el mismo entorno.

Gestión de Dependencias

Una parte importante del trabajo con entornos virtuales es la gestión adecuada de las dependencias:

Instalar desde requirements.txt

Cuando compartes un proyecto o trabajas en una nueva máquina:

Recrear entorno desde requirements.txt Terminal 
# 1. Clonar o descargar el proyecto
# 2. Crear nuevo entorno virtual:
python -m venv venv

# 3. Activar entorno virtual:
source venv/bin/activate  # Linux/WSL
# o
venv\Scripts\activate     # Windows

# 4. Instalar todas las dependencias:
pip install -r requirements.txt

# 5. Verificar instalación:
pip list

Tipos de archivos de dependencias

Diferentes archivos de requirements Terminal 
# requirements.txt - Dependencias de producción:
pip freeze > requirements.txt

# requirements-dev.txt - Dependencias de desarrollo:
pip install pytest black flake8  # herramientas de desarrollo
pip freeze > requirements-dev.txt

# requirements-base.txt - Solo dependencias principales (sin versiones fijas):
echo "requests" > requirements-base.txt
echo "numpy" >> requirements-base.txt
echo "pandas" >> requirements-base.txt

Actualizar dependencias

Mantener dependencias actualizadas Terminal 
# Ver paquetes desactualizados:
pip list --outdated

# Actualizar un paquete específico:
pip install --upgrade requests

# Actualizar pip:
pip install --upgrade pip

# Actualizar todas las dependencias (¡cuidado!):
pip freeze | cut -d'=' -f1 | xargs pip install --upgrade

# Regenerar requirements.txt después de actualizar:
pip freeze > requirements.txt
Cuidado con las actualizaciones

Actualizar todas las dependencias a la vez puede romper tu proyecto. Es mejor actualizar de una en una y probar que todo sigue funcionando.

Estructura de Proyecto Recomendada

Una estructura de proyecto bien organizada facilita el trabajo con entornos virtuales:

Estructura de proyecto típica File Structure 
mi_proyecto/
├── venv/                    # Entorno virtual (no versionar)
├── src/                     # Código fuente
│   ├── __init__.py
│   ├── main.py
│   └── utils.py
├── tests/                   # Pruebas
│   ├── __init__.py
│   └── test_main.py
├── docs/                    # Documentación
├── requirements.txt         # Dependencias de producción
├── requirements-dev.txt     # Dependencias de desarrollo
├── .gitignore              # Archivos a ignorar en git
├── README.md               # Descripción del proyecto
└── setup.py                # Configuración del paquete (opcional)

Archivo .gitignore importante

Nunca versiones el directorio del entorno virtual. Crea un archivo .gitignore:

Contenido de .gitignore Git 
# Entornos virtuales
venv/
env/
ENV/
.venv/

# Archivos de Python
__pycache__/
*.py[cod]
*$py.class
*.so

# Archivos de sistema
.DS_Store
Thumbs.db

# IDEs
.vscode/
.idea/
*.swp
*.swo

# Variables de entorno
.env
¿Por qué no versionar venv/?

El directorio del entorno virtual contiene rutas absolutas y archivos específicos del sistema. Es mejor que cada desarrollador cree su propio entorno usando requirements.txt.

Herramientas Avanzadas

Para proyectos más complejos, puedes usar herramientas más avanzadas:

pipenv - Combinando pip y virtualenv

Usando pipenv Terminal 
# Instalar pipenv:
pip install pipenv

# Crear proyecto con pipenv:
mkdir mi_proyecto_pipenv
cd mi_proyecto_pipenv

# Crear entorno e instalar paquetes:
pipenv install requests numpy

# Instalar dependencias de desarrollo:
pipenv install pytest --dev

# Activar entorno:
pipenv shell

# Instalar desde Pipfile:
pipenv install

# Generar requirements.txt:
pipenv requirements > requirements.txt

conda - Para ciencia de datos

Usando conda Conda 
# Crear entorno con conda:
conda create --name mi_proyecto python=3.11

# Activar entorno:
conda activate mi_proyecto

# Instalar paquetes:
conda install numpy pandas matplotlib jupyter

# Ver entornos:
conda env list

# Exportar entorno:
conda env export > environment.yml

# Crear entorno desde archivo:
conda env create -f environment.yml

Configuración de VS Code con Entornos Virtuales

Visual Studio Code tiene excelente soporte para entornos virtuales de Python:

Configuración automática

  1. Abre tu proyecto en VS Code
  2. Crea tu entorno virtual con python -m venv venv
  3. VS Code detectará automáticamente el entorno virtual
  4. Selecciona el intérprete: Ctrl+Shift+P → "Python: Select Interpreter"
  5. Elige el Python de tu entorno virtual
Configuración de VS Code (.vscode/settings.json) JSON 
{
    "python.defaultInterpreterPath": "./venv/bin/python",
    "python.terminal.activateEnvironment": true,
    "python.linting.enabled": true,
    "python.linting.pylintEnabled": true,
    "python.formatting.provider": "black",
    "python.testing.pytestEnabled": true,
    "files.exclude": {
        "**/__pycache__": true,
        "**/venv": true
    }
}
Terminal integrado

Cuando VS Code está configurado correctamente, el terminal integrado activará automáticamente tu entorno virtual cuando abras una nueva terminal.
