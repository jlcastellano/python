Pydantic

¿Qué es Pydantic?

Pydantic es una biblioteca de Python que se utiliza para validar datos y definir esquemas de forma sencilla usando las anotaciones de tipos de Python.

Explicación Simple

Imagina que tienes un formulario de registro de usuarios. Necesitas asegurarte de que el nombre sea un texto, la edad sea un número positivo y el email tenga formato válido. Pydantic hace todo esto automáticamente.

Ejemplo Práctico

Sin Pydantic (validación manual):

def crear_usuario(datos):
    # Tienes que validar todo manualmente
    if "nombre" not in datos:
        raise ValueError("Falta el nombre")
    if not isinstance(datos["nombre"], str):
        raise ValueError("El nombre debe ser texto")
    if len(datos["nombre"]) < 2:
        raise ValueError("El nombre es muy corto")
    if "edad" not in datos:
        raise ValueError("Falta la edad")
    if not isinstance(datos["edad"], int):
        raise ValueError("La edad debe ser un número")
    if datos["edad"] < 0:
        raise ValueError("La edad no puede ser negativa")
    # ... y así con cada campo
    
    return datos

Con Pydantic (validación automática):

from pydantic import BaseModel, EmailStr, Field

class Usuario(BaseModel):
    nombre: str = Field(min_length=2)
    edad: int = Field(ge=0)  # ge = greater or equal (mayor o igual)
    email: EmailStr
    activo: bool = True  # Valor por defecto

# Pydantic valida automáticamente
usuario = Usuario(
    nombre="Ana",
    edad=25,
    email="ana@gmail.com"
)

print(usuario.nombre)  # Ana
print(usuario.model_dump())  # Convierte a diccionario

¿Qué hace Pydantic automáticamente?

Característica Descripción
Validación de tipos Verifica que cada campo sea del tipo correcto
Conversión automática Convierte "25" a 25 si el campo es int
Valores por defecto Permite definir valores predeterminados
Campos obligatorios Los campos sin valor por defecto son obligatorios
Mensajes de error Genera errores claros si los datos son inválidos
Serialización Convierte objetos a diccionarios o JSON fácilmente

Ejemplo de Validación

from pydantic import BaseModel, Field, EmailStr
from typing import Optional

class Cliente(BaseModel):
    nombre: str = Field(min_length=3, max_length=50)
    email: EmailStr
    telefono: str = Field(pattern=r'^\d{9}$')  # Exactamente 9 dígitos
    edad: int = Field(ge=18, le=120)  # Entre 18 y 120
    notas: Optional[str] = None  # Campo opcional

# Esto funciona correctamente
cliente_valido = Cliente(
    nombre="María García",
    email="maria@email.com",
    telefono="612345678",
    edad=30
)

# Esto lanza un error automáticamente
try:
    cliente_invalido = Cliente(
        nombre="AB",           # Muy corto (mínimo 3)
        email="no-es-email",   # Formato inválido
        telefono="123",        # No tiene 9 dígitos
        edad=15                # Menor de 18
    )
except Exception as e:
    print(e)

¿Por qué se usa con FastAPI?

FastAPI usa Pydantic para validar automáticamente los datos que llegan a tu API.

from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class Producto(BaseModel):
    nombre: str
    precio: float
    stock: int

@app.post("/productos/")
def crear_producto(producto: Producto):
    # FastAPI + Pydantic validan automáticamente
    # Si los datos son inválidos, devuelve error 422
    return {"mensaje": f"Producto {producto.nombre} creado"}

Si alguien envía datos incorrectos a tu API, FastAPI (usando Pydantic) rechaza la petición automáticamente con un mensaje de error claro.
