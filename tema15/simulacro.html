Sistema de Comentarios y Ratings con FastAPI + Tkinter

PARTE 1: DESARROLLO DE LA API (40 puntos)

Requisito 1.1: Estructura y Modelos Pydantic (8 puntos)

Tarea: Crea los modelos Pydantic necesarios para la aplicación.

Especificaciones:

  • Modelo Comentario con campos: id, autor, contenido, rating, timestamp, producto_id
  • Modelo ComentarioCrear para validación de entrada (sin id ni timestamp)
  • Validación: rating debe estar entre 1 y 5
  • Validación: autor y contenido no deben estar vacíos
  • Validación: contenido debe tener mínimo 10 caracteres
  • Validación: autor máximo 50 caracteres

Requisito 1.2: Endpoints CRUD (20 puntos)

Tarea: Implementa los siguientes endpoints:

Método Ruta Descripción Puntos
POST /comentarios Crear comentario 4
GET /comentarios Listar todos 3
GET /comentarios/{id} Obtener uno 2
PUT /comentarios/{id} Actualizar 4
DELETE /comentarios/{id} Eliminar 4
GET / Endpoint raíz 3

Especificaciones:

  • POST: Recibe ComentarioCrear, retorna Comentario con id y timestamp automático
  • GET (lista): Retorna lista ordenada por timestamp descendente, opcionalmente filtrada por producto
  • GET (uno): Retorna comentario específico, 404 si no existe
  • PUT: Solo permite actualizar contenido y rating
  • DELETE: Retorna confirmación, 404 si no existe
  • Todas las respuestas en JSON válido

Requisito 1.3: Filtrado, Búsqueda y Estadísticas (12 puntos)

Tarea: Implementa los siguientes endpoints adicionales:

1. Filtrar por rating (4 puntos)

GET /comentarios/filtro/rating?min=3&max=5
  • Retorna comentarios con rating entre min y max (inclusive)
  • Validación: min y max entre 1-5
  • Ordenados por timestamp descendente

2. Filtrar por producto (4 puntos)

GET /comentarios/por-producto/{producto_id}
  • Retorna comentarios del producto especificado
  • Ordenados por rating descendente
  • 404 si producto no tiene comentarios

3. Estadísticas (4 puntos)

GET /comentarios/estadisticas/{producto_id}

Retorna JSON con:

  • Promedio de rating
  • Cantidad de comentarios
  • Rating mínimo y máximo
  • Distribución por stars (cuántos de 1, 2, 3, 4, 5)
  • 404 si producto sin comentarios

PARTE 2: INTERFAZ GRÁFICA CON TKINTER (40 puntos)

Requisito 2.1: Layout y Componentes (12 puntos)

Tarea: Diseña una interfaz Tkinter con los siguientes componentes:

Panel de Entrada (Nueva Pestaña):

  • Campo texto: Nombre del autor (validación: 2-50 caracteres)
  • Campo numérico: ID del producto (validación: número positivo)
  • Área texto multilínea: Contenido del comentario (mínimo 4 líneas visibles)
  • Selector de rating: 5 radio buttons con estrellas (1-5)
  • Botón: "Enviar Comentario"
  • Botón: "Limpiar Campos"
  • Indicador de carga mientras se envía

Panel de Visualización (Nueva Pestaña):

  • Tabla (Treeview) con columnas: ID | Autor | Rating | Contenido | Fecha | Producto
  • Barra de desplazamiento vertical y horizontal
  • Altura mínima 15 filas
  • Menú contextual (click derecho) con opciones: Eliminar, Ver detalles

Panel de Filtros:

  • Combobox para filtrar por rating (Todos, 5⭐, 4⭐, 3⭐, 2⭐, 1⭐)
  • Spinbox para filtrar por ID de producto
  • Botón "Filtrar"
  • Botón "Limpiar Filtros"
  • Etiqueta mostrando cantidad total y promedio de comentarios

Panel de Estadísticas (Nueva Pestaña):

  • Spinbox para seleccionar ID producto
  • Botón "Ver Estadísticas"
  • Etiquetas para: promedio, total, mínimo, máximo
  • Área texto para mostrar gráfico de distribución en ASCII

Requisito 2.2: Funcionalidad de la GUI (18 puntos)

Tarea: Implementa las siguientes funciones:

1. Crear comentario (4 puntos)

  • Validación: autor no vacío y 2-50 caracteres
  • Validación: contenido 10+ caracteres
  • Validación: rating 1-5
  • Validación: producto_id número positivo
  • Envío POST a API en thread separado
  • Mostrar mensaje de éxito/error
  • Limpiar campos automáticamente tras éxito
  • Actualizar tabla de comentarios

2. Listar y visualizar comentarios (4 puntos)

  • Al iniciar la aplicación, cargar comentarios automáticamente
  • Mostrar en tabla ordenados por fecha (descendente)
  • Cada fila contiene: ID, autor, rating (con estrellas), contenido resumido, fecha, producto
  • Click en fila muestra detalles completos
  • Actualizar tabla después de crear/eliminar

3. Filtrar comentarios (5 puntos)

  • Filtro por rating: mostrar solo comentarios con rating seleccionado
  • Filtro por producto: mostrar solo del producto especificado
  • Ambos filtros pueden combinarse
  • Botón "Limpiar Filtros" restaura vista completa
  • Actualiza tabla correctamente
  • Actualiza contador de resultados

4. Operaciones de eliminación (5 puntos)

  • Click derecho en fila abre menú contextual
  • Opción "Eliminar" visible
  • Confirmación antes de eliminar
  • Envío DELETE a API
  • Actualizar tabla automáticamente
  • Mostrar confirmación de éxito
  • Manejo de errores

Requisito 2.3: Threading, Errores y Experiencia de Usuario (10 puntos)

Tarea: Implementa gestión de excepciones robusta.

Especificaciones:

  • Indicador visual de carga (spinner o etiqueta "⏳ Cargando...")
  • Manejo de requests.exceptions.Timeout
  • Manejo de requests.exceptions.ConnectionError
  • Manejo de errores HTTP (400, 404, 500, etc.)
  • Mensajes de error claros y en español
  • Recovery automático después de errores
  • Los timouts de peticiones: máximo 5 segundos

PARTE 3: INTEGRACIÓN Y DOCUMENTACIÓN (20 puntos)

Requisito 3.1: Integración Completa (10 puntos)

Tarea: Asegura que API y GUI funcionan juntas correctamente.

Especificaciones:

  • API escucha en http://127.0.0.1:8000
  • GUI se conecta a URL correcta
  • Ambas pueden ejecutarse simultáneamente sin conflictos
  • Los datos persisten durante la sesión
  • Prueba exitosa con al menos 5 comentarios diferentes
  • Prueba exitosa: crear, listar, filtrar, estadísticas y eliminar
  • Documentación clara sobre cómo ejecutar ambas

Requisito 3.2: Documentación y Código Limpio (10 puntos)

Tarea: Proporciona documentación profesional y código bien estructurado.

1. README.md (5 puntos)

  • Descripción clara del proyecto
  • Requisitos de instalación (Python 3.8+, FastAPI, Tkinter, etc.)
  • Instrucciones paso a paso para instalar dependencias
  • Cómo ejecutar la API (comando exacto)
  • Cómo ejecutar la GUI (comando exacto)
  • Descripción breve de cada endpoint
  • Ejemplos de uso (curl o similar)
  • Estructura del proyecto explicada

2. Código bien comentado (5 puntos)

  • Docstrings en todas las funciones principales
  • Comentarios en lógica compleja
  • Type hints en todo el código
  • Variables con nombres descriptivos
  • Separación clara de secciones
  • Máximo 100 caracteres por línea
  • Uso de convenciones PEP 8

Comentarios

  • Contenido: 10-500 caracteres, no vacío
  • Rating: Número entero entre 1 y 5
  • Producto ID: Número positivo
